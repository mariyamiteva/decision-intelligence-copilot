# Decision Framework

This framework is used for structured evaluation of strategic and operational cases.

## Core principles
1. Prefer options that are supported by evidence from more than one source.
2. If the expected benefits are material but data quality is weak, the case should be marked as Needs Review.
3. If the proposal introduces high operational complexity without proportional value, it should not be recommended.
4. When trade-offs exist, the preferred option is the one with the clearest implementation path and measurable outcomes.
5. Missing critical information should be surfaced explicitly rather than assumed away.

## Confidence guidance
- Confidence above 0.80 should be used only when evidence is complete and aligned.
- Confidence between 0.60 and 0.79 indicates moderate support with known constraints.
- Confidence below 0.60 should usually correspond to review or rejection scenarios.
